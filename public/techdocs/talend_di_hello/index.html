  1. Create a new Job by right-clicking on Repository > Job Designs, then selecting Create Standard Job.
  2. Enter HelloWorld for the Name, then populate the Purpose and Description. Click Finish when done.

3 Start adding a component to the Job by clicking anywhere in the left-hand portion of the Designer. Begin typing the name of the component, in this case tFixedFlowInput, and a list of matching components will appear. Search the list for the tFixedFlowInput component. Notice the suffix Misc. This indicates the folder within the Palette in which the component can also be found.

  1. ADD THE COMPONENT Double-click the tFixedFlowInput component in the list to plant the component at the selected spot in the Designer.

The component is now added to the Job. This “click and type” method represents a quick and easy way to add components to your Jobs. In the next step, you will add a second component using an alternate method.

  1. ADD THE COMPONENT In the Palette view, begin typing the name of another component, in this case tLogRow, into the Find Component box and then click the Search button (). You can also press the Enter key.

The component has been found under the Logs & Errors folder. Select the tLogRow component in the Palette view, then click on a spot to the right of tFixedFLowInput_1 in the Designer to place the new component.

The component is now added to the Job. This approach represents another way you can browse components and add them to your Job. Note that it is also possible to simply drag and drop the component from the Palette to your Job’s design space.

  1. CONNECT THE COMPONENTS Right-click on the tFixedFlowInput component, then select Row > Main.

Move the mouse over top of tLogRow_1 component and click on it.

The link will be created and displayed. A row (or link) allows data to flow from one component to another. The arrow indicates the direction of the flow.

  1. Configure Components Double-click tFixedFlowInput_1 to open the Component view, which is located in the lower portion of the main window.

Then click on the […] button near Edit schema to configure the schema.

Click the Add button () to add a new column. Change the Column name to Message, then click OK.

Answer Yes when asked if you would like to propagate changes.

  1. PROVIDE A MESSAGE Notice that the Message column is added to the schema. Now, set the Value of the Message Column to “Hello World”.

Be sure to include the opening and closing quotation marks.

  1. SAVE THE JOB Note there is an asterisk (*) preceding the name of your Job in the Designer tab. This means the Job has changes which have not been saved yet. Click the Save button in the main toolbar () and the * disappears.

You can also press Ctrl + S to save the Job.

  1. OPEN THE RUN VIEW To run the Job, open the Run view (near the Component tab in the lower portion of the window) and click Run.

You can also use the Run icon () in the main toolbar to run the Job. REVIEW JOB OUTPUT Talend Studio saves the Job, builds it, and then runs it. This may take a few moments. Once complete, the Job runs and finishes quickly, displaying messages in the Run view.

You can see the Hello World message is then displayed in the execution console.

ADD A COMPONENT TO READ A CSV FILE In the Palette, click File > Input, then scroll down until you locate the component called tFileInputDelimited.

Components in the Palette are grouped by function. This particular component reads input from a delimited file. Select tFileInputDelimited in the Palette, and then click in the Designer to place the component. Click the button marked with an ellipsis […] next to the File Name / Stream field to browse for the input file.

Beispiel Datei von Open-Data-Portal der Deutsche Bahn http://data.deutschebahn.com

Übersicht Bahnhöfe RNI (Stand 01/2018) URL: http://download-data.deutschebahn.com/static/datasets/stationsdaten_regio/DBRNI-Uebersicht_Bahnhoefe-Stand2018-01.csv

CONFIGURE FIELD SEPARATOR AND HEADER PROPERTIES Still in the Component view, change the value in the Field Separator box to “,” to indicate that a comma separates column values in the file, rather than the default semicolon.

Then, change the value in the Header box to 1 to specify that the first row of the input file contains column names.

IND THE tMap COMPONENT Click anywhere in the Designer to the right of CustomersFile. Begin typing the name of the component you wish to add, in this case tMap, and a list of matching components will appear. Search the list for the component.

PLACE AND CONNECT THE COMPONENT Now, right-click Customers File and select Row > Main, then click tMap_1. This creates a connection between the two components.

A list of components are then suggested. Start typing the name of the desired output component, in this case tFileOutputDelimited, into the search box. As its name implies, a tFileOutputDelimited component writes rows to a file in delimited format.

OPEN THE MAPPING EDITOR Double-click tMap_1 to open the mapping editor.

The tMap component provides so much functionality, so this window has many elements. Notice the table on the left, labeled row2, and the table on the right labeled CappedOut. Those tables represent the schema of the input and output connections, with names that match the rows. The schema for row2 is copied from the input component and should look familiar. The schema for the new row CappedOut is not defined yet.

The goal of this Job is to pass all the data from row1 to CappedOut, while capitalizing the values in the State column.

COPY ALL INPUT ROWS In row2, click on the top row and then shift-click on the bottom row, to select all the rows. Then, drag the selected rows from row2 onto CappedOut.

Notice the arrows indicating the mapping of columns from one schema to the other. In the CappedOut table, notice the Expression, which defines the data content, as well as the column names. At this point, the data and column names are an exact match to those in row1 (the expression row1. specifies the data in the name column of row2).

SELECT THE OUTPUT YOU WISH TO MODIFY The goal is to convert the State column data to uppercase, so click row1.State in the CappedOut output table, and then click the button marked with an ellipsis […] that appears.

The Expression Builder opens. This window allows you to construct expressions in Java syntax. You can enter the expression as text yourself, or you can take advantage of the tool to simplify the building of such expressions.

BUILD AN EXPRESSION TO CONVERT DATA TO UPPERCASE Select Categories > StringHandling, then scroll down in the Functions list until you find UPCASE. Select it. Notice the text that appears in the Help area.

INSERT THE FUNCTION INVOCATION INTO THE EXPRESSION Double-click UPCASE to insert the function into the expression at the insertion point. StringHandling.UPCASE(row2.Bundesland)

Notice the format of the expression that is inserted. As a default example, the function contains the literal string “Hello”. You want to reconfigure this to convert the contents of the State column.

Fernverkehr & Nahverkehr typ von String auf boolean wechsen StringHandling.UPCASE(row2.Fernverkehr).equals(“JA”)?true:false
