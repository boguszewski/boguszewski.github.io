BW6 Domain

Ein AppSpace ist eine Gruppe von einem oder mehreren AppNodes. Der Appspace Ordner enthält zwei Unterordner - apps und shared. Der Apps-Ordner enthält alle Anwendungen, die in dem AppSpace deployed wurden.

AppNode Eine AppNode ist ein OSGI Container in der eine BW Anwendung ausgeführt wird. Jede Anwendung die deployed wurde läuft standardmäßig auf allen AppNodes des AppSpaces. Der AppNode Ordner enthält die ausführbare Binärdateien und entsprechend den .TRA Konfigurationsdateien, sowie die Log Files des AppNodes

Application Archive Ein Application Archive ist die Deployment Unit für eine Anwendung, die über das TIBCO Business Studio erzeugt wird. Ein Application Archiv enthält eine oder mehrere OSGi-Bundles, je eines für alle Module der Anwendung. Außerdem enthält es alle Metadaten der Anwendung.

BWAgent Zur Verwaltung läuft auf jeder Maschiene / jedem Computer der Domäne ein bwagent Deamon. Ein bwagent ist für die Bereitstellung von AppsNode und Anwendungen verantwortlich, für die Administration und die Synchronisation des Domain Datastores mit dem lokalen Filesystem. Bwagents kommunizieren miteinander über einen gemeinsamen Datenspeicher und ein Messaging System. Beispielsweise ActiveSpaces für Datenpersistenz und Kommunikation oder durch eine Datenbank für Datenpersistenz und TIBCO Enterprise Message Service für die Kommunikation.

Auf einen bwagent kann über mehrere Wege zuzugegriffen werden: Komandozeile - bwadmin WEB-Admin-UI - TEA Direct HTTP - REST-API

Die BusinessWorks6 Laufzeitumgebung ist hierarchisch aufgebaut. Als oberste Ebene beschreibt die Domain eine isolierte Laufzeitumgebung. Innerhalb einer Domain existieren ein oder mehrere AppSpaces die die Domain in logische Gruppen unterteilt. Applikationen werden in einen AppSpace deployed. In einem AppSpace existieren wiederum ein oder mehrere AppNodes in denen die im AppSpace deployeden Applikationen ausgeführt werden. Ein AppNode ist ein OSGI Container auf einer JVM. Eine Anwenung besteht aus ein oder mehreren OSGI Bundles für den Container. Es kann einzeln gesteuert werden, wie eine Anwenung in einem AppSpace mit mehreren AppNodes gestartetet wird.

Es gibt zwei Laufzeitoptionen für eine Domain Local und Enterprise. Im Local Mode wird die Domain nur auf einem Host instanziert und alle Konfigurationen werden auf in Files auf dem Host gespeichert. Im Enterprise Mode können Domain und AppSpace über mehrere Maschienen verteilt werden. Dazu läuft ein BWAgent auf den einzelnen Maschinen. Domain Konfigurationen können in ActiveSpaces oder in einer Datenbank persistiert werden. Die Kommunikation innerhalb einer Domain erfolg über ActiveSpaces, FTL oder EMS. Wobei ActiveSpaces der Default ist, aber nur für Testumgebungen verwendet werden sollte.

Konfiguriert und gesteuert werden können die Domain, AppSpacea, AppNodes und Anwendungen mit dem Commandline Tool bwadmin.

1) BWAdmin starten $ bwadmin

2) Mit einer neuen Commandozeile den BWagent starten $ bwagent
